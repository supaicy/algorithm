== 문제 설명
활동 선택 문제는 주어진 여러 활동 중에서 시작 시간과 종료 시간이 겹치지 않도록 최대한 많은 활동을 선택하는 문제입니다.

그리디 알고리즘은 각 단계에서 최선이라고 판단되는 선택을 반복하여 최종 해답에 도달하는 방식입니다. 이 문제에서는 **종료 시간이 빠른 활동**을 우선적으로 선택하여 최적의 결과를 도출합니다.

== 문제 해결 접근 방식
그리디 알고리즘의 접근 방식으로 활동 선택 문제를 해결하기 위해 다음 단계를 따릅니다:

1. **종료 시간 기준 정렬**: 활동들을 종료 시간이 빠른 순으로 정렬합니다.
2. **탐욕적 선택**: 가장 먼저 끝나는 활동을 선택하고, 그 후에는 시작 시간이 이전에 선택된 활동의 종료 시간보다 빠르지 않은 활동을 반복적으로 선택합니다.

이 방법을 통해 최대한 많은 활동을 선택할 수 있습니다.

== 알고리즘 설명
- **탐욕적 선택 속성 (Greedy Choice Property)**: 종료 시간이 가장 빠른 활동을 먼저 선택하면, 남아 있는 시간 동안 더 많은 활동을 선택할 가능성이 높아집니다.
- **최적 부분 구조 (Optimal Substructure)**: 각 단계에서 최선의 선택을 하면, 이 선택은 나머지 부분 문제에서도 최적의 해를 제공합니다.

== 실행 결과
활동 목록이 다음과 같이 주어졌을 때:
- (1, 3), (2, 5), (4, 6), (6, 7), (5, 9), (8, 9)

최적의 활동 선택 결과는 다음과 같습니다:
- (1, 3)
- (4, 6)
- (6, 7)
- (8, 9)

== 학습 내용
이 알고리즘은 그리디 접근 방식을 사용하여 문제를 해결하는데, 매 단계에서 최선의 선택을 반복적으로 수행하여 최종적인 최적의 해답에 도달합니다. 이 과정에서 탐욕적 선택이 전체에서도 최선의 결과를 제공함을 이해할 수 있습니다.
